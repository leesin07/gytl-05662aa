# 资金看板调整说明

## 📋 修改概述

根据用户需求，对资金及交易数据看板的计算逻辑和显示标签进行了调整。

---

## 🎯 修改内容

### 1. 总本金 → 总资产

#### 修改前
- **标签**：总本金
- **计算逻辑**：仅计算入金总额
- **公式**：`总本金 = 所有 deposit 类型的资金记录金额之和`

#### 修改后
- **标签**：总资产
- **计算逻辑**：入金 + 出金（负数） + 总盈利 + 总亏损（负数） - 手续费总额
- **公式**：`总资产 = 入金总额 + 出金总额 + 总盈利 + 总亏损 - 手续费总额`

**说明**：
- 入金总额：所有存入的资金总和（正数）
- 出金总额：所有取出的资金总和（负数）
- 总盈利：所有盈利交易的盈利总额（正数）
- 总亏损：所有亏损交易的亏损总额（负数）
- 手续费总额：所有交易的手续费总和（正数）
- 净资产 = 入金 + 出金 + 净收益
- 净收益 = 总盈利 + 总亏损 - 手续费总额

---

### 2. 当前余额 → 总市值

#### 修改前
- **标签**：当前余额
- **计算逻辑**：资金记录的最后一条余额
- **公式**：`当前余额 = 资金记录最后一条的 balance`

#### 修改后
- **标签**：总市值
- **计算逻辑**：当前未卖出的股票的总市值
- **公式**：`总市值 = 所有买入未卖出的股票数量 × 当前价格`

**重要说明**：
- 由于当前系统没有实时股价数据，**总市值暂时设置为 0**
- 如需实现真实的总市值计算，需要：
  1. 添加持仓管理功能，记录当前持有的股票和数量
  2. 集成实时股价数据（如东方财富 API）
  3. 定期更新持仓市值

## 📊 计算示例

### 示例场景

假设有以下交易记录：

| 操作 | 类型 | 金额 | 手续费 | 盈亏 |
|------|------|------|--------|------|
| 初始入金 | deposit | 100,000 | - | - |
| 买入股票A | buy | 10,000 | 5 | - |
| 卖出股票A（盈利） | sell | 12,000 | 6 | +1,989 |
| 出金 | withdraw | -5,000 | - | - |
| 买入股票B | buy | 20,000 | 10 | - |
| 卖出股票B（亏损） | sell | 19,000 | 9.5 | -1,019.5 |

### 计算过程

#### 总资产计算

```
入金总额 = 100,000
出金总额 = -5,000（注意：出金是负数）
总盈利 = 1,989
总亏损 = -1,019.5
手续费总额 = 5 + 6 + 10 + 9.5 = 30.5

总资产 = 100,000 + (-5,000) + 1,989 + (-1,019.5) - 30.5
       = 95,939
```

#### 总市值计算

```
假设没有持仓（所有股票都已卖出）：
总市值 = 0

如果有持仓：
- 持有股票C 100股，当前价格 10元
总市值 = 100 × 10 = 1,000
```

## 💡 使用说明

### 查看总资产

总资产反映了您的投资账户的总体状况：

- **总资产 > 入金总额**：整体盈利
- **总资产 < 入金总额**：整体亏损
- **总资产 = 入金总额**：盈亏平衡

### 查看总市值

总市值反映了当前持仓的价值：

- **总市值 = 0**：当前无持仓
- **总市值 > 0**：有持仓（需要实时数据支持）

---

## ⚠️ 注意事项

1. **总市值暂为 0**
   - 当前系统没有实时股价数据
   - 需要额外开发持仓管理和实时数据功能
   - 建议后续添加此功能以实现完整的资产管理

2. **手续费计算**
   - 手续费在总资产计算中是**减项**
   - 公式中的"手续费扣减"理解为"减去手续费总额"

3. **盈亏计算**
   - 总亏损在公式中是负数
   - 显示时取绝对值（正数）
