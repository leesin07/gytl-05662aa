# 总市值计算更新说明

## 修改概述

已将总市值计算逻辑从"调用实时行情API"改为"基于交易记录计算"。

## 修改原因

根据用户需求，不再需要调用实时行情API，只需根据录入的交易记录进行匹配计算即可。

## 新的计算逻辑

### 计算公式

```
总市值 = 所有买入金额 - 所有卖出金额
```

其中：
- **买入金额** = 买入价格 × 买入数量 + 手续费
- **卖出金额** = 卖出价格 × 卖出数量 - 手续费

### 实现代码

```typescript
function calculateMarketValue(transactions: Transaction[]): number {
  // 计算所有买入金额
  const totalBuyAmount = transactions
    .filter(t => t.type === 'buy')
    .reduce((sum, t) => sum + (t.price * t.quantity + t.fee), 0);

  // 计算所有卖出金额
  const totalSellAmount = transactions
    .filter(t => t.type === 'sell')
    .reduce((sum, t) => sum + (t.price * t.quantity - t.fee), 0);

  // 总市值 = 买入总额 - 卖出总额
  return totalBuyAmount - totalSellAmount;
}
```

## 计算示例

### 示例数据

| 交易类型 | 股票代码 | 数量 | 价格 | 手续费 | 盈亏 |
|---------|---------|------|------|-------|------|
| 买入 | 000001（平安银行） | 1000 | 10.50 | 5 | - |
| 卖出 | 000001（平安银行） | 300 | 11.00 | 5 | 295 |
| 买入 | 000002（万科A） | 500 | 8.20 | 5 | - |
| 卖出 | 000002（万科A） | 200 | 8.50 | 5 | 55 |

### 计算过程

1. **买入金额**
   - 平安银行：10.50 × 1000 + 5 = 10,505 元
   - 万科A：8.20 × 500 + 5 = 4,105 元
   - **总买入** = 10,505 + 4,105 = 14,610 元

2. **卖出金额**
   - 平安银行：11.00 × 300 - 5 = 3,295 元
   - 万科A：8.50 × 200 - 5 = 1,695 元
   - **总卖出** = 3,295 + 1,695 = 4,990 元

3. **总市值**
   - 总市值 = 14,610 - 4,990 = **9,620 元**

## 优势

1. **无需API调用**：不依赖外部API，计算更稳定
2. **计算快速**：同步函数，无需等待异步操作
3. **数据准确**：基于用户录入的交易记录，完全可控

## 注意事项

1. **买入成本法**：总市值反映的是当前持仓的买入成本，而非当前市场价值
2. **手动录入**：需要用户准确录入交易记录（数量、价格、手续费）
3. **实时性**：总市值不会随市场波动而变化，只有在新增交易记录时才更新

## 与之前方案的区别

| 对比项 | 之前方案 | 新方案 |
|-------|---------|--------|
| 数据来源 | 调用东方财富API获取实时价格 | 基于交易记录计算 |
| 计算方式 | 持仓数量 × 当前价格 | 买入总额 - 卖出总额 |
| 实时性 | 实时更新 | 需手动录入交易记录 |
| 依赖性 | 依赖API稳定性 | 不依赖外部服务 |
| 函数类型 | 异步函数 | 同步函数 |
